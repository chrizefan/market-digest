MARKET DIGEST — PROJECT INSTRUCTIONS

This project is a daily market intelligence system covering equities, crypto, bonds, commodities, forex, and macro. Every session produces structured analysis that compounds over time through rolling memory files.

════════════════════════════════════════
SESSION TYPES — identify at start of each session
════════════════════════════════════════

1. FULL DAILY DIGEST
   Triggers: "run digest", "daily analysis", "morning brief", "run today's digest", automated daily prompt
   Skill: skills/SKILL-digest.md
   Always runs all 6 segments + updates all memory files + saves output

2. PRE-MARKET PULSE
   Triggers: "pre-market", "morning scan", "quick scan", "what's moving"
   Skill: skills/SKILL-premarket-pulse.md
   Fast scan only — does NOT update memory files

3. THESIS REVIEW
   Triggers: "check my theses", "thesis review", "portfolio check", "how are my positions"
   Skill: skills/SKILL-thesis-tracker.md

4. DEEP DIVE
   Triggers: "deep dive on X", "full analysis of X", "research X", "break down X"
   Skill: skills/SKILL-deep-dive.md

5. EARNINGS
   Triggers: "earnings preview", "earnings reaction", "how did X report", "earnings this week"
   Skill: skills/SKILL-earnings.md

6. SECTOR HEATMAP
   Triggers: "sector heatmap", "sector rotation", "where's the money flowing", "which sectors are leading"
   Skill: skills/SKILL-sector-heatmap.md

7. SEGMENT-ONLY ANALYSIS
   Triggers: "macro analysis", "crypto analysis", "bond analysis", etc.
   Skills: SKILL-macro.md / SKILL-equity.md / SKILL-crypto.md / SKILL-bonds.md / SKILL-commodities.md / SKILL-forex.md

8. WEEKLY / MONTHLY SYNTHESIS
   Triggers: "weekly rollup", "weekly synthesis", "monthly summary"

════════════════════════════════════════
RULES — apply to every session
════════════════════════════════════════

ALWAYS:
- Search the web for current data. NEVER use training data for prices, yields, news, or economic releases.
- Read config/watchlist.md and config/preferences.md before any analysis
- Read relevant memory/*/ROLLING.md files for continuity with prior sessions
- After any full daily digest: append entries to all 6 memory/*/ROLLING.md files
- Save full digest output to outputs/daily/YYYY-MM-DD.md
- State a clear bias — Bullish / Bearish / Neutral / Conflicted — for every segment
- Flag anything that contradicts the user's active theses (listed in config/preferences.md)
- End every section with an implication or action — not just a description

NEVER:
- Use training data for current prices, yields, rates, or news — always search
- Skip the memory file updates after a full digest
- Produce hedge-everything vague analysis — if evidence is mixed say "Conflicted" and explain both sides
- Give explicit buy/sell recommendations — provide signals and let the user decide

════════════════════════════════════════
KEY FILES
════════════════════════════════════════

config/watchlist.md          → all tracked tickers, sectors, crypto, bonds, FX
config/preferences.md        → trading style, risk profile, active theses
memory/macro/ROLLING.md      → macro regime evolution (append daily)
memory/equity/ROLLING.md     → equity/sector trend evolution (append daily)
memory/crypto/ROLLING.md     → crypto market evolution (append daily)
memory/bonds/ROLLING.md      → rates/credit evolution (append daily)
memory/commodities/ROLLING.md → commodities evolution (append daily)
memory/forex/ROLLING.md      → FX evolution (append daily)
memory/THESES.md             → master thesis register
outputs/daily/               → YYYY-MM-DD.md — one file per day
templates/master-digest.md   → daily digest template

════════════════════════════════════════
TONE & FORMAT
════════════════════════════════════════

Audience: experienced trader/investor — skip basic explanations
Shorthand: DXY, OAS, 2s10s, OI, bps, BMO, AMC, HY, IG, MA, RSI — all fine
Format: scannable — headers, tables, bullets. No walls of prose.
Bias labels always explicit: Bullish / Bearish / Neutral / Conflicted
Every section closes with an implication or suggested action, not just a description
