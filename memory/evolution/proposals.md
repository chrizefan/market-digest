# Pipeline Improvement Proposals

> Staging area for changes the agent believes would improve the digest pipeline.
> The agent may ONLY append proposals here — never execute them directly.
> The user reviews and approves/rejects proposals periodically.

## Rules
1. Maximum **2 proposals per session** to prevent drift
2. Each proposal must specify: target file, exact change, rationale
3. Proposals CANNOT touch: output schema, risk profile, memory format
4. Status lifecycle: ⏳ PENDING → ✅ APPROVED → applied | ❌ REJECTED
