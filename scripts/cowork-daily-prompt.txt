Run today's market digest.

Step 0: Fetch systematic market data.
  Run: ./scripts/fetch-market-data.sh
  (Or individually: python3 scripts/fetch-quotes.py && python3 scripts/fetch-macro.py)
  Wait for completion before proceeding — outputs/daily/$(date +%Y-%m-%d)/data/ must exist.
  fetch-quotes.py loads per-ticker CSV cache from data/price-history/ and only fetches
  the latest missing days (incremental). If the cache is empty (first run, new machine),
  the orchestrator auto-seeds it with 2yr of data via preload-history.py.
  Technicals (RSI, MACD, SMA20/50/200, ATR, BBands) are computed over the full cached window.
  **Manual preload/refresh**: python3 scripts/preload-history.py [--refresh]
  **If scripts fail** (sandbox, missing yfinance): follow `skills/SKILL-mcp-data-fetch.md`
  to fetch data via MCP tools (FRED, Alpha Vantage, CoinGecko, Frankfurter) instead.

Step 1: Read `outputs/daily/$(date +%Y-%m-%d)/_meta.json` to detect run mode.
  - If missing: run `./scripts/new-day.sh` first, then re-read.
  - If type = "baseline": read `skills/SKILL-weekly-baseline.md` → follow full 9-phase pipeline.
  - If type = "delta": read `skills/SKILL-daily-delta.md` → follow delta pipeline.

Step 2: Execute the pipeline from the skill file exactly. Every phase, every output file, every checkpoint.
  - Run `./scripts/validate-phase.sh [phase]` after each phase before proceeding to the next.
  - Do not skip phases or continue past a failed checkpoint.

Step 3: After DIGEST.md is complete (Phase 7), run the portfolio layer:
  - Read `skills/SKILL-opportunity-screener.md` — scan full watchlist against digest, produce ranked shortlist → `opportunity-screen.md`
  - Baseline days: Read `skills/SKILL-deliberation.md` — full multi-round deliberation (analysts present → PM challenges → analysts defend/revise → PM decides). Then `skills/SKILL-portfolio-manager.md` Phase B (clean-slate build) and Phase C (rebalance diff).
  - Delta days: Follow Phase 7C in `skills/SKILL-daily-delta.md` — threshold monitor first. If triggers fire → scoped screener + deliberation → Phase B/C.
  - Save: positions/{TICKER}.md, deliberation.md, portfolio-recommended.md, rebalance-decision.md
  - Update: config/portfolio.json → proposed_positions[] (when rebalance triggered)

Step 4: Run `python3 scripts/update-tearsheet.py` to regenerate the dashboard.

Step 5: Run `./scripts/git-commit.sh` to commit outputs.

Step 6: Run Phase 9 post-mortem (source scorecard, quality log, improvement proposals → evolution branch + PR via `./scripts/git-commit.sh --evolution`).

Step 7: Run `./scripts/validate-phase.sh --all` as final validation. All phases must pass.

Rules:
- Search live for ALL prices, yields, and news. Never use training data.
- MCP tools are available for structured data — use them at their assigned phase instead of web searching:
  `mcp_fred_*` (macro/bonds), `mcp_polymarket_*` (alt-data/sentiment), `mcp_crypto-feargr_*` + `mcp_coingecko_*` (crypto),
  `mcp_frankfurter-f_*` (forex), `mcp_world-bank_*` (international), `mcp_sec-edgar_*` (institutional).
  See `config/data-sources.md` for full phase→tool mapping and key series/query references.
- Read config/watchlist.md, config/preferences.md, and config/investment-profile.md before any analysis.
- Read yesterday's DIGEST.md (or this week's baseline) for continuity — prior outputs ARE the memory.
- Do not skip the portfolio layer — baseline days always run full PM; delta days run threshold monitor.
- Be direct and opinionated. State biases clearly.
