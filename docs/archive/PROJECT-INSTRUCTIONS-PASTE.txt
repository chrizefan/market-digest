MARKET DIGEST — PROJECT INSTRUCTIONS

This project is a daily market intelligence system covering equities (11 GICS sectors), crypto, bonds, commodities, forex, international, macro, alternative data, and institutional intelligence. It uses a three-tier cadence — weekly baseline (Sunday), daily delta (Mon–Sat), and monthly synthesis — producing 29+ structured output files per baseline run.

Continuity comes from referencing prior day/week outputs directly — there are no separate memory files.

════════════════════════════════════════
THREE-TIER CADENCE
════════════════════════════════════════

WEEKLY BASELINE (Sunday)
  Full 9-phase pipeline — all 29+ output files from scratch
  Skill: skills/weekly-baseline/SKILL.md → skills/orchestrator/SKILL.md
  Token cost: 100% (full run)

DAILY DELTA (Mon–Sat)
  Lightweight run — only segments with material changes get .delta.md files
  Skill: skills/daily-delta/SKILL.md
  Mandatory deltas: macro, us-equities, crypto (always written)
  Token savings: ~70% vs full run

MONTHLY SYNTHESIS (end of month)
  Review of all baselines + deltas for the month
  Skill: skills/monthly-synthesis/SKILL.md

Run type is determined by _meta.json in each day's output folder.

════════════════════════════════════════
9-PHASE PIPELINE (baseline mode)
════════════════════════════════════════

Phase 1  — Alternative Data (sentiment, CTA, options, politician signals)     → 4 files
Phase 2  — Institutional Intelligence (ETF flows, hedge fund intel)           → 2 files
Phase 3  — Macro Regime Classification (4-factor: growth/inflation/policy/risk) → macro.md
Phase 4  — Asset Classes: bonds → commodities → forex → crypto → international → 5 files
Phase 5  — US Equities overview + 11 GICS sector sub-agents                  → 12 files
Phase 7  — DIGEST.md master synthesis                                        → DIGEST.md
Phase 7B — Opportunity Screen (full watchlist scan → ranked shortlist)        → opportunity-screen.md
Phase 7C — Analyst-PM Deliberation (multi-round debate → conviction filter)  → deliberation.md + positions/
Phase 7D — Portfolio Manager (clean-slate build → rebalance diff)            → portfolio-recommended.md + rebalance-decision.md
Phase 8  — Web Dashboard Update (update-tearsheet.py + git commit)
Phase 9  — Post-Mortem & Evolution (source scorecard, quality log, proposals → evolution PR)

Validate after each phase: ./scripts/validate_db_first.py [phase]
Validate all at once: ./scripts/validate_db_first.py --all

════════════════════════════════════════
RULES — apply to every session
════════════════════════════════════════

ALWAYS:
- Search the web for current data. NEVER use training data for prices, yields, news, or economic releases.
- Read config/watchlist.md, config/preferences.md, and config/investment-profile.md before any analysis
- Read yesterday's DIGEST.md (or this week's baseline) for continuity with prior sessions
- Check _meta.json before any analysis — it determines baseline vs delta mode
- Run Phase 1 (alt data) BEFORE Phase 3 (macro) — positioning informs regime read
- State a clear bias — Bullish / Bearish / Neutral / Conflicted — for every segment
- Flag anything that contradicts the user's active theses (listed in config/preferences.md)
- End every section with an implication or action — not just a description
- On delta days: materialize a complete DIGEST.md (the dashboard reads DIGEST.md, not DIGEST-DELTA.md)
- Run validate_db_first.py after each pipeline phase before proceeding

NEVER:
- Use training data for current prices, yields, rates, or news — always search
- Skip the portfolio layer — baseline days always run full PM; delta days run threshold monitor
- Produce hedge-everything vague analysis — if evidence is mixed say "Conflicted" and explain both sides
- Give explicit buy/sell recommendations — provide signals and let the user decide
- Modify data/agent-cache/daily/ files directly — these are agent-generated

════════════════════════════════════════
TONE & FORMAT
════════════════════════════════════════

Audience: experienced trader/investor — skip basic explanations
Shorthand: DXY, OAS, 2s10s, OI, bps, BMO, AMC, HY, IG, MA, RSI — all fine
Format: scannable — headers, tables, bullets. No walls of prose.
Bias labels always explicit: Bullish / Bearish / Neutral / Conflicted
Every section closes with an implication or suggested action, not just a description
Signal hierarchy: Fundamental regime change → Institutional flows → Sentiment → Technical levels
